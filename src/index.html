About Accessibility

Web accessibility refers to the practice of designing and developing websites and web applications in a way that ensures people with disabilities can use them effectively. This includes individuals with visual, auditory, motor, or cognitive impairments. The goal is to make web content perceivable, operable, understandable, and robust for everyone, regardless of their abilities or how they interact with the web.

Color Contrast

Color contrast is crucial for web accessibility because it ensures that text and important elements are easily distinguishable, especially for users with visual impairments like color blindness or low vision.

  • The WCAG (Web Content Accessibility Guidelines) defines three levels of color contrast ratios—A, AA and AAA. You should always aim for AA (4.5:1) or AAA (7:1).
  • Use tools to check if your color combinations meet accessibility standards
  • Don’t use color as the only means of conveying important information
  • Read more about color contrast!

Alternative text

Alternative text (alt text) is a brief description added to images in HTML that helps screen readers convey the content of the image to visually impaired users. It also appears if the image fails to load. More about alternative text.

Be Descriptive

Provide clear and concise descriptions relevant to the context.

Skip Decorative Images

Use empty alt text (alt="") for purely decorative images so they are ignored by screen readers.

Search Engine

Alt text provides better image context/descriptions to search engine crawlers, helping them to index and rank an image properly in image search.

Semantic HTML

Semantic HTML uses HTML tags that convey meaning about the content they enclose, making the structure of a webpage more understandable for both browsers and developers. For example, tags like header, nav, section and footer describe the purpose of the content, rather than using generic tags like div or span. This improves accessibility, SEO, and code readability.

Read more about semantic HTML.

ARIA

Accessible Rich Internet Applications is a set of attributes that enhance the accessibility of web content, especially for users who rely on assistive technologies like screen readers. ARIA helps make dynamic content and complex UI elements more understandable by describing their roles, states, and properties.

  • ARIA Roles define what an element is or how it should be perceived by assistive technologies (role="button").
  • ARIA States describe the current condition of an interactive element. These are dynamic and can change (aria-expanded="true").
  • ARIA Properties give additional information about an element. Unlike states, properties are typically static. (aria-label="Close")
  • ARIA helps describe UI elements that may not have clear semantic meaning in HTML (like custom dropdowns, modals, etc.).
  • Whenever possible, use standard HTML elements (like button, nav, header) since they are accessible by default.
  • ARIA is meant to enhance accessibility where native HTML lacks features, not as a substitute for semantic HTML. Read more about Aria here.
